# Security Policy

## Supported Versions

Security fixes are provided on a best-effort basis for the latest development
line on `main` and the most recent tagged release.

## Reporting a Vulnerability

Please do not open public GitHub issues for security-sensitive findings.

Send a private report with:

- affected version or commit
- impact summary
- reproduction steps or proof of concept
- any suggested mitigation

Until a dedicated security mailbox is added, use GitHub private security
reporting if enabled for the repository. If private reporting is not available,
contact the maintainer directly before public disclosure.

## Disclosure Expectations

- We aim to acknowledge valid reports within 7 days.
- We aim to provide a remediation plan or status update within 14 days.
- Please allow reasonable time for a fix before public disclosure.

## Scope Notes

`cobol-intel` can be used in regulated environments, but repository examples,
benchmarks, and local scripts are not a substitute for your own production
security review, deployment hardening, and data handling controls.
